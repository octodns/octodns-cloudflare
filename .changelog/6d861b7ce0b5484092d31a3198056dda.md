CloudflareProvider now manages a record's Cloudflare Regional Services (Data Localization) region via `octodns.cloudflare.region` on `A`/`AAAA`/`CNAME`/`ALIAS` records — read on populate, and created/changed/removed on apply. Cloudflare applies a region per-hostname and only to proxied records, so conflicting or non-proxied usage is reported via `strict_supports`. See #143.
